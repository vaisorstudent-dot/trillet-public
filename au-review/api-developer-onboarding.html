Deploy Your First Voice Agent

A six-step guide for agency partners and integration developers

This guide walks through building and deploying a production-ready inbound voice agent via the Trillet API. Each step is gated: complete the verification before moving to the next step. A failed verification means the next step will not work. By the end, you'll have a live phone number routing inbound calls to an AI agent with post-call SMS and analytics.

Steps in this guide
  1. Prerequisites and authentication
  2. Step 1: Create the voice agent
  3. Step 2: Create the call flow
  4. Step 3: Configure knowledge base
  5. Step 4: Provision a phone number
  6. Step 5: Wire the phone to the agent
  7. Step 6: Test and verify end-to-end
  8. Common issues

Prerequisites and Authentication

Before starting, you need four things:

API Key
Generate at app.trillet.ai > Settings > Developer > API Keys
Workspace ID
Found in the workspace URL or in your JWT access_token payload
An existing call flow to duplicate
Flow creation via POST is broken. You need at least one flow in your workspace to duplicate
curl or a REST client
Examples use curl. Any HTTP client works

Set your credentials as environment variables to avoid repeating them in every call:

export TRILLET_API_KEY="your-api-key-here"
export TRILLET_WORKSPACE_ID="your-workspace-id-here"
export TRILLET_BASE="https://api.trillet.ai"

Every request needs two headers:

-H "x-api-key: $TRILLET_API_KEY"
-H "x-workspace-id: $TRILLET_WORKSPACE_ID"
Python note: All inline Python examples use sys.stdout.reconfigure(encoding='utf-8'). This is required on Windows to avoid encoding errors with non-ASCII characters in API responses. Include it in every Python snippet.
Step 1 of 6
Create the voice agent
Goal: A configured voice agent with your chosen LLM, TTS voice, and STT provider. Save the returned _id as AGENT_ID.

The agent defines the AI brain and voice. The call flow (Step 2) defines the script and behavior. You need both.

RESULT=$(curl -s -X POST "$TRILLET_BASE/v1/api/agents" \
  -H "x-api-key: $TRILLET_API_KEY" \
  -H "x-workspace-id: $TRILLET_WORKSPACE_ID" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Inbound Receptionist",
    "llmModel": "gpt-4.1",
    "ttsModel": {
      "provider": "rime",
      "voiceId": "mistv2_Brook",
      "language": "en"
    },
    "sttProvider": {
      "id": "deepgram-nova-3"
    },
    "settings": {
      "speed": 1.0,
      "volume": 1.0,
      "temperature": 0.7
    },
    "smartCallback": true,
    "maxAttemptsForSmartCallback": 3,
    "gapBetweenEachAttemptForSmartCallback": [
      {"days": 0, "hours": 1},
      {"days": 0, "hours": 4},
      {"days": 1, "hours": 0}
    ]
  }')

AGENT_ID=$(echo "$RESULT" | python -c "
import json, sys
sys.stdout.reconfigure(encoding='utf-8')
print(json.load(sys.stdin)['_id'])
")
echo "Agent ID: $AGENT_ID"

Step 2 of 6
Create the call flow
Goal: A configured call flow attached to your agent, with a prompt, welcome message, and call settings. Save the returned _id as FLOW_ID.
Known Bug: Direct flow creation is broken
POST /v1/api/call-flows returns 500 on all payloads as of March 2026. The workaround is to duplicate an existing flow, then configure it via PUT. You need the ID of any existing flow in your workspace. Find one with GET /v1/api/call-flows and use its _id as SOURCE_FLOW_ID.

Step 2a: Duplicate an existing flow

SOURCE_FLOW_ID="paste-any-existing-flow-id-here"

DUP_RESULT=$(curl -s -X POST "$TRILLET_BASE/v1/api/call-flows/$SOURCE_FLOW_ID/duplicate" \
  -H "x-api-key: $TRILLET_API_KEY" \
  -H "x-workspace-id: $TRILLET_WORKSPACE_ID" \
  -H "Content-Type: application/json" \
  -d '{}')

FLOW_ID=$(echo "$DUP_RESULT" | python -c "
import json, sys
sys.stdout.reconfigure(encoding='utf-8')
print(json.load(sys.stdin)['_id'])
")
echo "Flow ID: $FLOW_ID"

Step 2b: Configure the duplicated flow

Write your prompt to a file to avoid shell escaping issues with quotes and special characters. PUT uses replace semantics: the entire object below replaces the duplicated flow's config. 

cat > /tmp/flow_payload.json << 'PAYLOAD'

  "prompt": "You are a professional receptionist for Acme Corp. Your job is to answer questions about our services, take messages, and transfer callers who need to speak with someone urgently.\n\nServices we offer:\n- General consulting\n- Technical support\n- Account management\n\nIf someone needs urgent assistance, transfer them to the owner. Otherwise, take their name and number and let them know someone will call back within one business day.",
  "welcomeMessage": "ai_custom",
  "customWelcomeMessage": "Thank you for calling Acme Corp. How can I help you today?",
  "agent": "REPLACE_WITH_AGENT_ID",
  "settings": {
    "transfers": [
      {
        "friendly_name": "Owner",
        "method": "cold",
        "destination": "+1XXXXXXXXXX",
        "destinationType": "SIP",
        "condition": "Caller requires urgent assistance or requests a human",
        "isSummary": false,
        "summary_format": "",
        "custom_headers": []
      }
    ],
    "callSetting": {
      "maxCallDuration": 600,
      "endCallOnSilence": 15,
      "enableComplianceMonitoring": true,
      "voicemailDetection": {"enabled": true, "action": "hangup"}
    },
    "speechSetting": {
      "responsiveness": 7,
      "interruptionSensitivity": 5
    },
    "OfficeHoursSettings": {
      "enabled": false
    },
    "smsPostCallSettings": {
      "businessOwnerSMS": true,
      "customerSMS": false,
      "useInternalA2P": true,
      "selectedPhoneNumberId": "",
      "businessOwnerPhoneNumber": "+1XXXXXXXXXX"

# Replace the placeholder with your actual agent ID
sed -i "s/REPLACE_WITH_AGENT_ID/$AGENT_ID/" /tmp/flow_payload.json

curl -s -X PUT "$TRILLET_BASE/v1/api/call-flows/$FLOW_ID" \
  -H "x-api-key: $TRILLET_API_KEY" \
  -H "x-workspace-id: $TRILLET_WORKSPACE_ID" \
  -H "Content-Type: application/json" \
  -d @/tmp/flow_payload.json
Transfer schema: Use friendly_name and destination, not label and phoneNumber. The older field names are accepted by PUT but not read by the transfer_call function. The agent will report "Team not found" at transfer time.
Verify before continuing
curl -s "$TRILLET_BASE/v1/api/call-flows/$FLOW_ID" \
  -H "x-api-key: $TRILLET_API_KEY" \
  -H "x-workspace-id: $TRILLET_WORKSPACE_ID" | python -c "
import json, sys
sys.stdout.reconfigure(encoding='utf-8')
d = json.load(sys.stdin)
assert d['name'] == 'Inbound Receptionist', f'Name: {d[\"name\"]}'
assert d['agent'] == '$AGENT_ID' or d.get('agent',{}).get('_id') == '$AGENT_ID', 'Agent not set'
assert len(d['prompt']) > 50, 'Prompt too short or missing'
sms = d.get('settings',{}).get('smsPostCallSettings',{})
assert sms.get('useInternalA2P'), 'useInternalA2P not set -- SMS will silently fail'
print(f'PASS: Flow {d[\"_id\"]} configured correctly')
"
Step 3 of 6
Configure a knowledge base (optional but recommended)
Goal: A knowledge base with business-specific content attached to the call flow. Skip this step if you don't need KB-backed responses.

Step 3a: Create the KB

KB_RESULT=$(curl -s -X POST "$TRILLET_BASE/v1/api/knowledgebase" \
  -H "x-api-key: $TRILLET_API_KEY" \
  -H "x-workspace-id: $TRILLET_WORKSPACE_ID" \
  -H "Content-Type: application/json" \
  -d '{"name": "Acme Corp Info", "description": "Business information for the receptionist"}')

KB_ID=$(echo "$KB_RESULT" | python -c "
import json, sys
sys.stdout.reconfigure(encoding='utf-8')
print(json.load(sys.stdin)['_id'])
")
echo "KB ID: $KB_ID"

Step 3b: Add content

curl -s -X POST "$TRILLET_BASE/v1/api/knowledgebase/$KB_ID/text" \
  -H "x-api-key: $TRILLET_API_KEY" \
  -H "x-workspace-id: $TRILLET_WORKSPACE_ID" \
  -H "Content-Type: application/json" \
  -d '{
    "text": "Acme Corp provides consulting services for small and medium businesses. We are open Monday through Friday, 9am to 6pm Eastern. Our main office is at 123 Main Street. For billing questions, callers should ask for the accounts team. For technical issues, our support line is available during business hours. Common questions: we do not offer weekend appointments; our typical project turnaround is 2-3 weeks; we accept payment by invoice, credit card, or ACH transfer."
  }'

Step 3c: Attach KB to the flow

GET the full flow first, add the KB ID, then PUT the complete object:

FLOW_DATA=$(curl -s "$TRILLET_BASE/v1/api/call-flows/$FLOW_ID" \
  -H "x-api-key: $TRILLET_API_KEY" \
  -H "x-workspace-id: $TRILLET_WORKSPACE_ID")

# Add KB ID to knowledgeBases array and PUT the updated flow
echo "$FLOW_DATA" | python -c "
import json, sys
sys.stdout.reconfigure(encoding='utf-8')
d = json.load(sys.stdin)
d.setdefault('settings', {}).setdefault('knowledgeBases', [])
if '$KB_ID' not in d['settings']['knowledgeBases']:
    d['settings']['knowledgeBases'].append('$KB_ID')
print(json.dumps(d))
" | curl -s -X PUT "$TRILLET_BASE/v1/api/call-flows/$FLOW_ID" \
  -H "x-api-key: $TRILLET_API_KEY" \
  -H "x-workspace-id: $TRILLET_WORKSPACE_ID" \
  -H "Content-Type: application/json" \
  -d @-
File upload: The KB file upload endpoint (POST /knowledgebase/{id}/files) works via browser but the correct curl multipart field name is unconfirmed. Text upload via /text is the reliable programmatic path.
Step 4 of 6
Provision a phone number
Goal: A Trillet-managed phone number in your workspace. Save the returned _id as PHONE_ID.

Step 4a: Check your quota

curl -s "$TRILLET_BASE/v1/api/twilio/free-number-status" \
  -H "x-api-key: $TRILLET_API_KEY" \
  -H "x-workspace-id: $TRILLET_WORKSPACE_ID"
# Returns: {"limit": 5, "used": 3, "remaining": 2, "hasFree": true}

Step 4b: Search for available numbers

curl -s "$TRILLET_BASE/v1/api/twilio/available-numbers?country=US&type=local&areaCode=212" \
  -H "x-api-key: $TRILLET_API_KEY" \
  -H "x-workspace-id: $TRILLET_WORKSPACE_ID"
# Returns array: [{phoneNumber, friendlyName, region, capabilities: {voice, SMS, MMS}}]

Step 4c: Purchase a number

PHONE_RESULT=$(curl -s -X POST "$TRILLET_BASE/v1/api/twilio/purchase-number" \
  -H "x-api-key: $TRILLET_API_KEY" \
  -H "x-workspace-id: $TRILLET_WORKSPACE_ID" \
  -H "Content-Type: application/json" \
  -d "{
    \"country\": \"US\",
    \"phoneNumber\": \"+12125550100\",
    \"type\": \"local\",
    \"workspaceId\": \"$TRILLET_WORKSPACE_ID\"
  }")

PHONE_ID=$(echo "$PHONE_RESULT" | python -c "
import json, sys
sys.stdout.reconfigure(encoding='utf-8')
print(json.load(sys.stdin)['_id'])
")
echo "Phone ID: $PHONE_ID"
Verify before continuing
curl -s "$TRILLET_BASE/v1/api/twilio/user-phone-numbers" \
  -H "x-api-key: $TRILLET_API_KEY" \
  -H "x-workspace-id: $TRILLET_WORKSPACE_ID" | python -c "
import json, sys
sys.stdout.reconfigure(encoding='utf-8')
numbers = json.load(sys.stdin)
ids = [n['_id'] for n in numbers]
assert '$PHONE_ID' in ids, f'Phone ID not found in workspace. IDs: {ids}'
print('PASS: Phone number confirmed in workspace')
"
Step 5 of 6
Wire the phone to the agent
Goal: Inbound calls to the phone number route to your agent's call flow. This is the critical step. Without it, inbound calls get a disconnected tone.

Binding sets the phone number's pathwayId to your agent's flow and flowType to "call". Two calls are needed: bind the phone to the agent, then set the agent's pathway to the flow.

Step 5a: Bind the phone number to the agent

curl -s -X POST "$TRILLET_BASE/v1/api/agents/$AGENT_ID/bind/$PHONE_ID" \
  -H "x-api-key: $TRILLET_API_KEY" \
  -H "x-workspace-id: $TRILLET_WORKSPACE_ID" \
  -H "Content-Type: application/json" \
  -d '{
    "recordCalls": false,
    "recordingPublic": false,
    "officeHours": {"enabled": false},
    "metadataTags": {},
    "label": "",
    "transferCallerIdMode": "from-transferee"
  }'

Step 5b: Set the agent's pathway to the flow

# GET the full agent first to avoid wiping other fields
AGENT_DATA=$(curl -s "$TRILLET_BASE/v1/api/agents/$AGENT_ID" \
  -H "x-api-key: $TRILLET_API_KEY" \
  -H "x-workspace-id: $TRILLET_WORKSPACE_ID")

# Merge pathway into the full agent object and PUT it back
echo "$AGENT_DATA" | python -c "
import json, sys
sys.stdout.reconfigure(encoding='utf-8')
d = json.load(sys.stdin)
d['pathway'] = '$FLOW_ID'
print(json.dumps(d))
" | curl -s -X PUT "$TRILLET_BASE/v1/api/agents/$AGENT_ID" \
  -H "x-api-key: $TRILLET_API_KEY" \
  -H "x-workspace-id: $TRILLET_WORKSPACE_ID" \
  -H "Content-Type: application/json" \
  -d @-
Verify before continuing
curl -s "$TRILLET_BASE/v1/api/twilio/user-phone-numbers" \
  -H "x-api-key: $TRILLET_API_KEY" \
  -H "x-workspace-id: $TRILLET_WORKSPACE_ID" | python -c "
import json, sys
sys.stdout.reconfigure(encoding='utf-8')
numbers = json.load(sys.stdin)
phone = next((n for n in numbers if n['_id'] == '$PHONE_ID'), None)
assert phone, 'Phone not found'
assert phone.get('pathwayId') == '$FLOW_ID', f'pathwayId is {phone.get(\"pathwayId\")}, expected $FLOW_ID'
assert phone.get('flowType') == 'call', f'flowType is {phone.get(\"flowType\")}'
print(f'PASS: Phone {phone[\"phoneNumber\"]} is routed to flow $FLOW_ID')
"
Step 6 of 6
Test and verify end-to-end
Goal: A live call handled by the agent, verified in call history with transcript and post-call analytics.

Step 6a: Make a test call

Dial the phone number you purchased from any phone. You should hear the welcome message within 2-3 seconds. Have a short conversation with the agent, then hang up.

Alternatively, initiate an outbound test call via API:

curl -s -X POST "$TRILLET_BASE/v1/api/call" \
  -H "x-api-key: $TRILLET_API_KEY" \
  -H "x-workspace-id: $TRILLET_WORKSPACE_ID" \
  -H "Content-Type: application/json" \
  -d "{
    \"to\": \"+1YOUR_PHONE_NUMBER\",
    \"call_agent_id\": \"$AGENT_ID\",
    \"dynamic_variables\": {\"caller_name\": \"Test Caller\"}
  }"
Office hours
If office hours were enabled in the flow configuration, outbound calls placed outside the window sit in pending status indefinitely. Check OfficeHoursSettings.enabled in your flow config, or call during the configured hours.

Step 6b: Verify call history

# Wait 30-60 seconds after the call ends for post-call processing
curl -s "$TRILLET_BASE/v2/api/call-history?limit=5" \
  -H "x-api-key: $TRILLET_API_KEY" \
  -H "x-workspace-id: $TRILLET_WORKSPACE_ID" | python -c "
import json, sys
sys.stdout.reconfigure(encoding='utf-8')
d = json.load(sys.stdin)
calls = d.get('data', [])
if not calls:
    print('No calls found. Wait another 30 seconds.')
    sys.exit(1)
latest = calls[0]
print(f'Call ID: {latest[\"_id\"]}')
print(f'Status: {latest[\"status\"]}')
print(f'Duration: {latest.get(\"duration\", \"N/A\")}s')
print(f'Transcript lines: {len(latest.get(\"transcript\", []))}')
print(f'Summary: {latest.get(\"summary\", \"(not yet generated)\")[:100]}')
"

Step 6c: Get full call detail with analytics

CALL_ID="paste-call-id-from-above"
curl -s "$TRILLET_BASE/v2/api/call-history/$CALL_ID" \
  -H "x-api-key: $TRILLET_API_KEY" \
  -H "x-workspace-id: $TRILLET_WORKSPACE_ID" | python -c "
import json, sys
sys.stdout.reconfigure(encoding='utf-8')
d = json.load(sys.stdin)
analyzed = d.get('analyzedData', {})
print('=== Call Analytics ===')
print(f'Sentiment: {analyzed.get(\"call_sentiment\")}')
print(f'Disposition: {analyzed.get(\"call_disposition\")}')
compliance = analyzed.get('compliance', {})
print(f'Compliance: {compliance.get(\"compliance_result\")}')
print(f'Cost: \${d.get(\"cost\", 0):.4f}')
print(f'Transcript lines: {len(d.get(\"transcript\", []))}')
print()
print('=== Full Transcript ===')
for t in d.get('transcript', [])[:5]:
    print(f'[{t[\"role\"]}] {t[\"content\"][:80]}')
"
Your deployment is complete when all of these pass
  • Call appears in GET /v2/api/call-history
  • Transcript array is non-empty
  • analyzedData exists with sentiment and disposition
  • Post-call SMS delivered (if enabled)
  • Post-call analysis email received (if configured)

Common Issues

Symptom Cause Fix
Inbound call gets disconnected tone Phone number not bound to agent (pathwayId not set) Run Step 5 again. Verify pathwayId on the phone number
Outbound call stays in "pending" forever Office hours are enabled, call placed outside the window Disable OfficeHoursSettings or call within the configured hours
Flow config changes don't appear after PUT PUT on a non-owned flow silently succeeds but writes nothing Verify the flow was created/duplicated by your API key user. Check via GET after every PUT
Config fields reset to defaults after PUT PUT uses replace semantics. Partial objects wipe omitted fields GET the full object first, merge your changes, PUT the complete object
POST /v1/api/call-flows returns 500 Known platform bug. Direct flow creation is broken Use the duplicate+configure workaround in Step 2
Transfer at runtime says "Team not found" Using old transfer field names (label/phoneNumber instead of friendly_name/destination) Update transfer entries to use UI format: friendly_name, destination, method, destinationType
SMS not delivered after call Missing useInternalA2P: true or businessOwnerPhoneNumber field Confirm all three required SMS fields are set: businessOwnerSMS, useInternalA2P, businessOwnerPhoneNumber
No transcript in call history Post-call processing takes 30-60 seconds after call ends Wait and re-GET. Transcript is filled asynchronously
"No active pathway configured" on web call start Agent's pathway field is not set PUT the agent with pathway: FLOW_ID (Step 5b)
