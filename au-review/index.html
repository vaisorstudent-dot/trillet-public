Key Findings for AU Review
1
Agency activation requires undocumented endpoints. Two calls critical to deploying an inbound agent — POST /agents/{id}/bind/{phoneId} and phone number purchase — are absent from the official docs entirely. A developer starting from docs.trillet.ai will fail before routing a single call.
2
All PUT endpoints use replace semantics, not merge. Sending a partial update wipes omitted fields silently with a 200 response. This is the root cause of the "settings resetting" failure mode. Fix: GET the full object first, merge, then PUT the complete object back.
3
Call flow creation via POST is broken (500 on all payloads). Workaround is confirmed: duplicate an existing flow, then configure via PUT. The UI uses this same approach internally.
4
Post-call SMS works; on-demand SMS does not. smsPostCallSettings with useInternalA2P: true delivers reliably. conversations/message/generate returns success but never delivers.
5
A full inbound deployment is achievable today via API. The 6-step gated workflow (agent → KB → flow → phone → bind → test) is documented, verified, and copy-paste ready in the Developer Onboarding document. The workarounds for all blocking bugs are included.
